# Emby质量监控插件

MoviePilot 插件 - 监控Emby媒体库中的电影质量，自动识别不达标资源并批量创建洗版订阅。

## 功能特性

- ✅ **质量监控**：扫描Emby媒体库，解析电影质量信息
- ✅ **智能判断**：根据分辨率、编码、来源、HDR等多维度判断质量
- ✅ **批量订阅**：一键批量创建MP洗版订阅
- ✅ **定时扫描**：支持Cron表达式配置定时任务
- ✅ **通知推送**：扫描完成后自动推送通知

## 质量判断维度

| 维度 | 说明 |
|------|------|
| 分辨率 | 720p / 1080p / 2160p (4K) |
| 视频编码 | H.264 / H.265 / AV1 |
| 来源类型 | WEB-DL / BluRay / REMUX |
| HDR信息 | SDR / HDR / Dolby Vision |

## 工作流程

```
扫描Emby媒体库 → 解析质量信息 → 质量判断 → 展示不达标列表 → 批量创建订阅
```

## 安装方法

### 方式1：通过MP插件市场

1. 进入MP设置 → 插件市场
2. 搜索"Emby质量监控"
3. 点击安装

### 方式2：手动安装

1. 下载本插件目录 `emby_quality_monitor`
2. 复制到MP插件目录：`app/plugins/`
3. 重启MP或开启插件热加载

## 配置说明

### 基本配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| Emby服务器 | 从MP已配置的Emby服务器中选择 | - |
| 媒体库名称 | 要监控的Emby媒体库名称 | 电影 |
| 最低分辨率 | 低于此值的电影将被标记 | 1080p |
| 优先编码 | 不在此列表的编码将被标记 | h265,hevc,av1 |
| 最低来源 | 低于此值的来源将被标记 | BluRay |
| 要求HDR | 开启后SDR电影将被标记 | 关闭 |
| 定时扫描 | Cron表达式，如 `0 2 * * *` | - |
| 开启通知 | 扫描完成后推送通知 | 开启 |

### 重要提示：配置MP整理覆盖模式

洗版成功后，旧文件的处理方式需要在MoviePilot中配置：

**路径：** MP设置 → 目录 → 整理模式 → 覆盖模式

- 选择 **"仅保留最新版本"** = 自动删除旧版本
- 选择 **"从不覆盖"** = 新旧版本共存

## API 接口

插件提供两个API接口：

### 1. 扫描媒体库

```bash
GET /api/v1/plugin/EmbyQualityMonitor/scan
```

**返回示例：**
```json
{
  "success": true,
  "message": "扫描完成，发现 3 部电影质量不达标",
  "data": [
    {
      "title": "沙丘2",
      "year": "2024",
      "tmdb_id": "823464",
      "current_quality": {
        "resolution": "1080p",
        "codec": "h264",
        "source": "WEB-DL",
        "hdr": "SDR"
      },
      "issues": [
        "编码非优选: h264",
        "来源不足: WEB-DL → BluRay"
      ]
    }
  ]
}
```

### 2. 批量创建订阅

```bash
POST /api/v1/plugin/EmbyQualityMonitor/subscribe
Content-Type: application/json

{
  "movies": [
    {
      "title": "沙丘2",
      "year": "2024",
      "tmdb_id": "823464"
    }
  ]
}
```

## 使用示例

### 场景1：手动扫描

1. 配置好插件参数
2. 勾选"立即运行一次"
3. 保存配置
4. 查看通知或日志，获取不达标列表

### 场景2：定时扫描

1. 配置Cron表达式（如 `0 2 * * *` 表示每天凌晨2点）
2. 开启"启用插件"
3. 系统自动定时扫描并推送通知

### 场景3：API调用

通过API集成到其他工作流：
- 定期扫描并自动创建订阅
- 与外部工具联动
- 批量处理媒体库
